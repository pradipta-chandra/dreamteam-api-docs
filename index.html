Dreamteam CRM API

A single, generic record API for every CRM object - contacts, deals, companies, meetings, notes, and custom objects - all served through one contract.

Authentication

All requests require an OAuth2 bearer token. The organization is resolved from the token's orgId claim - no tenant header or subdomain needed.

Authorization: Bearer <YOUR_TOKEN>

How to get your token:

Section in progress - details will be added once the token generation flow is available in the product.

Pagination

All list endpoints return paginated results using page (zero-based) and page_size (max 200, default 20).

Response metadata shape 
{
  "results": [ ... ],
  "metadata": {
    "page": 0,
    "page_size": 20,
    "total_elements": 137,
    "total_pages": 7,
    "has_next": true,
    "has_previous": false
  }
}

Records

CRUD, filter, and batch operations over any object type. Replace {object} with the type slug (e.g., contact, deal, company).

List Records

GET /api/v1/objects/{object}/records

Returns a page of records the caller is authorized to view. RBAC row-level scoped.

ParameterTypeDefaultDescription
pageinteger0Zero-based page index
page_sizeinteger20Page size (max 200)
sortstring-Sort spec, e.g. created_at,desc
associationsstring[]-Object types to embed (repeatable)
curl example 
curl --location 'https://api.dreamteamcrm.ai/api/v1/objects/contact/records?page=0&page_size=20&sort=created_at,desc' \
  --header 'Accept: application/json' \
  --header 'Authorization: Bearer <TOKEN>'

Create a Record

POST /api/v1/objects/{object}/records

Creates one record. Schema validation, dedup checks, and lifecycle hooks run in the write transaction.

curl example 
curl --location 'https://api.dreamteamcrm.ai/api/v1/objects/contact/records' \
  --header 'Accept: application/json' \
  --header 'Content-Type: application/json' \
  --header 'Authorization: Bearer <TOKEN>' \
  --data '{
    "properties": {
      "first_name": "Jane",
      "last_name": "Okafor",
      "email": "jane@acme.com",
      "owner_id": 42
    },
    "associations": [
      { "id": 456, "kind": "company" }
    ]
  }'

Get a Record by ID

GET /api/v1/objects/{object}/records/{id}

Fetch one record. Use ?associations=contact&associations=company to embed linked records (capped at 100 per type).

curl example 
curl --location 'https://api.dreamteamcrm.ai/api/v1/objects/contact/records/123' \
  --header 'Accept: application/json' \
  --header 'Authorization: Bearer <TOKEN>'
Sample response 
{
  "id": 123,
  "type": "contact",
  "properties": {
    "first_name": "Jane",
    "last_name": "Okafor",
    "email": "jane@acme.com",
    "owner_id": 42
  },
  "associations": {},
  "created_at": "2026-06-25T12:00:00Z",
  "updated_at": "2026-06-25T12:05:00Z",
  "created_by": 7
}

Replace a Record

PUT /api/v1/objects/{object}/records/{id}

Full replace. Fields the schema allows but the payload omits are cleared. For partial edits, use PATCH.

Partially Update a Record

PATCH /api/v1/objects/{object}/records/{id}

Only the fields you send get changed. Omitted fields stay untouched. To clear a field, send it with null.

Delete a Record

DELETE /api/v1/objects/{object}/records/{id}

Soft-delete. Lifecycle hooks may veto (e.g., a stage still referenced by deals). Returns 204.

Filter Records

POST /api/v1/objects/{object}/records/filter

Compound filtering. Filter groups are OR'd; filters within a group are AND'd (configurable). Supports lookup paths (owner_id.email) and association paths (@contact_company.name).

Filter operators
OperatorValue fieldDescription
EQUALSvalueExact match
NOT_EQUALSvalueNot equal
CONTAINSvalueSubstring match
NOT_CONTAINSvalueDoes not contain
STARTS_WITHvaluePrefix match
ENDS_WITHvalueSuffix match
GREATER_THANvalueGreater than
LESS_THANvalueLess than
GREATER_THAN_OR_EQUALSvalueGreater than or equal
LESS_THAN_OR_EQUALSvalueLess than or equal
INvaluesValue in list
NOT_INvaluesValue not in list
IS_NULLnoneField is empty
IS_NOT_NULLnoneField is not empty
BETWEENvalues (2)Range match (inclusive)
NOT_BETWEENvalues (2)Outside range
curl example - open deals over 10k 
curl --location 'https://api.dreamteamcrm.ai/api/v1/objects/deal/records/filter' \
  --header 'Accept: application/json' \
  --header 'Content-Type: application/json' \
  --header 'Authorization: Bearer <TOKEN>' \
  --data '{
    "filter_groups": [
      {
        "logical_operator": "AND",
        "filters": [
          { "field_path": "status", "operator": "EQUALS", "value": "open" },
          { "field_path": "amount", "operator": "GREATER_THAN", "value": 10000 },
          { "field_path": "owner_id", "operator": "EQUALS", "value": 100 }
        ]
      }
    ],
    "sort": { "field": "created_at", "direction": "DESC" },
    "pagination": { "page": 0, "page_size": 20 }
  }'

Batch Operations

POST /api/v1/objects/{object}/records/batch

Process up to 100 records in one call. Operations: create, read, update, upsert, delete. Set all_or_none: true for transactional mode.

Associations

Record-to-record links. The embed cap on record responses is 100 per type - use these endpoints to page beyond that or manage links directly.

List Association Links

GET /api/v1/objects/{object}/records/{id}/associations/{target_type}

Page through a record's links of one associated object type. Returns {id, type} entries only (no record data).

Link Two Records

PUT .../{id}/associations/{target_type}/{target_id}

Idempotent. Ensures the link exists. Send {"type": "primary_contact"} to pin a labeled association.

Unlink Two Records

DELETE .../{id}/associations/{target_type}/{target_id}

Idempotent. Removes the link if present. Use ?type= to remove only a specific labeled association.

Bulk Link/Unlink

POST .../{id}/associations/{target_type}/batch

Up to 1000 IDs per call. Transactional - any invalid target fails the whole batch.

Schema

Describe an Object Type

GET /api/v1/objects/{object}

Returns the effective schema - field names, types, whether required, filterable, searchable. Use before creating or updating records.
